ELEMENTO REACT  

COMPONENTE: 

¿Dónde lo usas en este problema?
R: Son un tipo de "piezas" de las páginas que cambian en tiempo real.
Por qué es el adecuado?
R: Porque en este tipo de páginas, se requieren cambios constantes, por ejemplo, el estado de una carta.

JSX: 

¿Dónde lo usas en este problema?

R: Es la "combinación" entre JAVASCRIPT Y HTML, una forma de producir código de forma más versátil gracias a las herramientas y operadores de JS.

Por qué es el adecuado?

R: Bastante útil para ejecutar los componentes cambiantes de este tipo de páginas.


PROPS: 

¿Dónde lo usas en este problema?

R: Son como las propiedades de un componente. Podría ser el nombre, su descripción, etc.

Por qué es el adecuado?

R: Para una página que se base en poder filtrar, buscar o ver las características de sus datos, este elemento funciona para diferenciarlos entre sí.

ESTADO (UseState): 

¿Dónde lo usas en este problema?

R: Es para considerar el tipo de estado de un factor de la página, por ejemplo: que esté en uso o no.

Por qué es el adecuado?

R: Es el adecuado al ser el tipo de elemento que se puede cambiar "a tiempo real" en páginas de este tipo.

RENDERIZADO DE LISTAS: 

¿Dónde lo usas en este problema?

R: Es para presentar/pasar los datos de un componente.

Por qué es el adecuado?

R: Útil para pasar datos de un componente a otro.

RENDERIZADO CONDICIONAL: 

¿Dónde lo usas en este problema?

R: Similar al anterior, solo que consiste en condiciones, como lo dice su nombre (por ejemplo: una cantidad de X cosa).

Por qué es el adecuado?

R: Útil y necesario en escenarios en donde se requiere efectuar algo en caso de que surja una condición específica.

¿Qué ventaja tiene dividir el recetario en componentes en lugar de escribir todo en un solo archivo?

R: Al dividirlo en 3 componentes, se posee la ventaja de que ninguno de ellos está sobrecargado de información, si se elimina uno por accidente aún quedarán los demás, se pueden construir para tareas más específicas y así pasarse su data en caso de ser necesario.

¿Qué diferencia hay entre props y estado en tu solución? Da un ejemplo concreto de cada uno tomado de esta aplicación.

R: Props es utilizado de forma más específica para los componentes de esta clase de páginas. Mientras que "estado" puede referirse a cómo se encuentra todo el código en sí, no sólo una variable en cuestión de algún componente, aunque también se podría considerar algo similar en ciertos escenarios en donde este término se utlice para preguntar sobre el estado de una propiedad.
